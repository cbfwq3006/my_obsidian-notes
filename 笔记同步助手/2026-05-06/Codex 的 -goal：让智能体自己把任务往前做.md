公众号名称：边跑步边读书

作者名称：苏耀勇

发布时间：2026-05-03 18:13

最近 Codex CLI 里有一个新命令：`/goal`。

它可以简单理解成：

**你给 Codex 一个目标，它会围绕这个目标持续执行，而不是只回答你一句话。**

以前我们用 Codex，更多是这样：

你让它改代码，它改一下。

你让它跑测试，它跑一下。

测试报错了，你再让它继续修。

修完之后，你还要提醒它再验证。

`/goal` 想解决的，就是这个来回指挥的问题。

你可以直接写：

​

/goal 修复当前项目里的登录问题，并确保相关测试通过

接下来，它会自己尝试执行：看代码、改代码、跑命令、看报错、继续修。

Codex 不只是“写代码工具”，它更像一个本地智能体。接入项目之后，它可以做很多工程里的杂活，比如修 bug、整理脚本、调整配置，也可以辅助修 OpenClaw、爱马仕这类项目里的问题。

`/goal` 相当于给这个智能体加了一个更明确的任务循环：

**不是做一步，而是朝一个目标持续推进。**

## 怎么用更稳

用 `/goal` 的关键，不是命令本身，而是目标怎么写。

不要只写：

​

/goal 优化这个项目

这太大，也太模糊。

更建议写成：

​

/goal 修复 OpenClaw 启动时报错的问题，优先保证服务能正常启动，不要改无关模块

或者：

​

/goal 修复hermes里当前失败的测试，限制最多循环 5 轮，最后给出修改文件和验证结果

一个比较好的 `/goal`，最好包含三件事：

​

## 1\. 要解决什么问题

## 2\. 怎么判断完成

## 3\. 范围和限制是什么

目标越清楚，Codex 越不容易跑偏。

## 适合什么场景

`/goal` 不只是用来修一个小 bug。

普通的小修改，不用 `/goal`，Codex 也能完成。

它更适合那些需要连续推进的任务：

• 需要读代码、改文件、跑命令、看报错

• 需要反复执行和验证

• 目标比较清楚，但过程可能要试几轮

• 你不想一直手动提醒它“继续”

但它不适合一上来就做这种事：

​

/goal 从 0 做一个完整系统

这种目标太大，边界太宽，很容易消耗大量额度，也容易改偏。

更稳的做法，是先把大任务拆小，再交给 `/goal`。

## 使用前先看额度

还有一个现实问题：额度。

`/goal` 会自己循环执行，一轮不够就继续下一轮。它比普通对话更容易消耗额度。

所以最好在 5 小时额度比较充足的时候用。

如果任务比较大，可以在目标里直接限制：

​

最多循环 5 轮。如果还没完成，先停下来总结当前进度和下一步。

这样做比较安全。

否则额度中途用完，它可能就停在半路，后面的工作没法继续。

我查了一下目前 Codex 的相关实现：`goals` 是持久化目标，官方代码里也有“continue over time”的描述。但这不等于额度用完后它一定会自动接着跑。

更稳妥的是：

**额度用完后，目标状态可能还在；等额度恢复后，你最好回到同一个 Codex 线程里，手动让它继续。不要完全依赖它自动续跑。**

## 为什么你可能看不到 `/goal`

我自己测试时，一开始没有看到 `/goal`。

需要先把 Codex 升级到 `0.128` 或以上版本。

如果你是用 npm 安装的，可以用：

​

npm install -g @openai/codex@latest

升级后，再看一下版本：

​

codex --version

版本在0.128以上才可以，然后打开配置。

在 Codex 的配置文件 `config.toml` 里加入：

​

goals = true

保存后重新打开 Codex，再输入 `/`，一般就能看到 `/goal`。

简单说，就是两步：

​

## 1\. 升级 Codex 到 0.128 或以上

## 2\. 在 config.toml 里加入 goals = true

## 还是要谨慎使用

开启之后，它会提示这是 under development feature。

![[笔记同步助手/images/34814760d009b30bb9d5d9d0d06c7654_MD5.png]]

也就是说，这还是一个开发中的功能。

我今天做了一些事情，但是还没有想好哪个不大不小的任务合适交给他。

## 最后

`/goal` 这个命令，让 Codex 往前迈了一大步。

它不只是多了一个斜杠命令，而是让 Codex 更像一个能持续推进任务的智能体。

人工智能的发展速度，确实比很多人想象得更快。现在它已经能读项目、改代码、跑命令、修错误。未来它会不会变成更成熟的工程助手，甚至能长期接管一部分重复性的开发工作？
