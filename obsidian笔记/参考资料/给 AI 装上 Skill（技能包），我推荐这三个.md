你有没有过这种感觉 —— 教了 AI 半天「我写文章喜欢短句、不要用赋能这种词」，下次开新对话，它又全忘了。

上次让它写周报写得挺好的，这次换了种理解方式，写出来完全不是那个味。

更头疼的是，你好不容易摸索出一套让 AI 高效干活的方法，想分享给同事，发现只能口述半天，对方还不一定能复现。

我被这几件事折磨了很久，直到发现了一个叫 Skill 的东西 —— AI 的「技能包」。

装上之后，AI 写的东西画风直接变了。随便看个例子：

装之前：

「在当今快速发展的技术领域中，人工智能正在深刻地改变着我们的工作方式。值得注意的是，AI 不仅仅是一个工具——它更是一种全新的思维范式。」

装之后：

「AI 改变了一些工作方式，但没宣传得那么夸张。我们团队用了半年，快的地方确实快了，但遇到复杂逻辑，AI 给的方案经常似是而非。说「推动可持续发展」太空了，不如说：能用的场景就好好用，不能用的别硬上。」

同一个 AI，同一个问题，区别就在于装没装那个「技能包」。

那这个技能包（Skill）到底是什么？

Skill 就是一份写给 AI 的工作手册，把你的经验装进它脑子里，让 AI 在特定场景下按照你预先设置好的流程和标准来给你干活。

打个比方。

![](https://mmbiz.qpic.cn/mmbiz_png/8jbeWJiaoDexg7BLgALJ7MBFOSN23SqxR47Ug27du0ertx80q6YmAkia7hud3UHbVfSB7QrSyHicL68NQk1XA8ETobJvUj6enoic1Kiaib75PTeh4/640?wx_fmt=png&from=appmsg)

AI 就像一个天赋很好的厨师，什么菜都能做，但每次全凭手感。有时候做得惊艳，有时候咸得要死。

给它一份你奶奶的手写菜谱——两勺酱油、三颗冰糖、小火炖 40 分钟——每次做出来都是那个味儿。

Skill 就是那份菜谱。

它其实就是一个文本文件，里面写着你希望 AI 遵守的流程和标准。AI 读到这个文件，就自动按里面的规则做事。而且这个 Skill 不挑工具，Cursor、Claude Code、Trae 都能用。

为什么需要它？

AI 有三个毛病，Skill 就是来治这三个的。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/8jbeWJiaoDezeMu32185DUJibpPw0lz8BRVehMsownkhcdS8lkeAWH6tSEfpVEicFc34TC4cGI9bI4baBp5fnbib4Qnl14G9nygcH5AoM1iavmVA/640?wx_fmt=png&from=appmsg)

1. 像金鱼一样，每次对话都失忆

你今天花 20 分钟教会 AI「我写文章喜欢短句、不要用赋能这种词、结尾要有行动项」，明天新开一个对话，它全忘了。你又得重新教一遍。

有了 Skill，教一次就够了。它不会忘。

2. 不够「靠谱」，同样的事发挥不稳定

你说「帮我写个周报」，这次写得很好，下次换个方式理解你的意思，写出来完全不是那个味。

有了 Skill，不管第几次，它都按同一个标准做。

3. 你的经验很难「复制」给别人

你摸索出了一套让 AI 高效干活的方法，但这套经验只在你脑子里。同事想学，你得口述半天，还不一定能复现。

有了 Skill，你的经验就是一个文件，发给别人直接就能用。

说白了，Skill 就是把你和 AI 磨合出来的经验，从脑子里抄出来，变成一份谁都能用的文件。

如果要安装 Skill，我首先推荐这三个

![](https://mmbiz.qpic.cn/mmbiz_png/8jbeWJiaoDeyq8gvg6hsfDzhI5AQHa9nj4DdSq7Vyh8dYwKeFaGriaYTIcicsFfBAogMrgsaWWVlKklhoYngLicQXYw93kiczoW24Fj0fH21ibQYI/640?wx_fmt=png&from=appmsg)

第一个：Humanizer——去 AI 味

AI 写的东西有一股特殊的味道，这个技能包可以让你的 AI 去「AI 味儿」。

比如，AI 爱说「值得注意的是」，喜欢三件套排比，结尾一定要升华一把，措辞永远两边不得罪。这个 Skill 能把这些痕迹全部干掉。

我们来用事实说话，用真实的文字演示一下。

改之前：

「异性频繁找你聊天，通常是一个强烈的信号，但也并不绝对等同于「喜欢」。要判断对方的真实意图，不能只看频率（次数），更要看聊天的内容、时间段以及互动的状态。」

改之后：

「异性老找你聊天，是不是喜欢你？大概率是。但别急着下结论。你试着两天不主动找他。如果他来了，多半有意思。如果没来，那之前可能就是无聊。」

![](https://mmbiz.qpic.cn/sz_mmbiz_png/8jbeWJiaoDez09UBAgyGvjCd3PYmCPS3L0lk6HFk0uPPvzRDcmt6BTFE71E5j1NhvfWrZ1wgvg3ZDLjZbibntkeU92Jdjcc8zN7OPxDYic3gng/640?wx_fmt=png&from=appmsg)

区别在哪？

原文两边都不得罪，通篇没有立场。

改完之后：先亮态度，给你一个能马上验证的办法。像个真人在跟你说话，而不是一篇分析报告。

再来一段。

改之前：

「午后的阳光斜斜地洒进窗台，猫咪蜷缩在那片金色的光斑里，尾巴偶尔轻轻晃动。远处传来孩子们的笑声，混合着卖冰淇淋小贩的叮当铃声。老槐树下，几位老人正在下棋，旁观者比对弈者还要着急，不时发出「哎呀」的叹息声。这就是夏天的味道——慵懒、温热，带着一点点甜。」

改之后：

「下午在阳台上坐着，猫占了唯一那块有太阳的地方，赶都赶不走。楼下有人在吵架，好像是下棋的那几个老头，天天吵，谁输了都不服。旁边卖冰淇淋的车又来了，铃声一响，小孩从各个方向冒出来，跑得比谁都快。热得不想动。但也不讨厌这种热。」

![](https://mmbiz.qpic.cn/mmbiz_png/8jbeWJiaoDewxGmibIUGVa6nmUmRO8j3CctI6gzHUlH92CwibwcXt6GH0WmhhpTgD5ibAXSkBdsNwmDZVWudo32pAF5XTlxhoQ4Gh5lbQEEAQ8A/640?wx_fmt=png&from=appmsg)

注意最后一句。

原文是「这就是夏天的味道——慵懒、温热，带着一点点甜」，破折号加三件套总结，AI 最爱的收尾方式。

真正的人写东西，不会在最后帮你下定义。

感受是读出来的，不是写出来的。

为什么第一个就推荐这个？

因为你把改之前和改之后往那一放，不用解释，对方自己就能看出区别。

那这个技能怎么装？

不管你用什么 AI 工具，安装 Skill 都很简单。基本原理都是把你从网上下载的或者自己写的 Skill 放到 ~/.xxx/skills 文件夹下，如果是 Cursor，就是 ~/.cursor/skills 文件夹；如果是 Claude，就是  ~/.claude/skills  文件夹。

要么直接打开命令行，一行命令就搞定：

```
git clone https://github.com/blader/humanizer.git ~/.claude/skills/humanizer
```

如果不知道什么是命令行，也没关系。去 GitHub 仓库把 SKILL.md 文件下载下来，手动放到对应的 skills 文件夹里，效果一样。

装好之后怎么用？

装完之后，你正常让 AI 帮你写东西就行。AI 会自动识别到这个 Skill，按里面的规则来写。

你也可以直接跟 AI 说「帮我把这段文字去一下 AI 味」，AI 就知道该怎么做了。

第二个：Find Skills——找更多技能包

到这里，相信你已经知道 Skill 有多好用了。那你肯定想：还有没有其他的？

但每个人的工作不一样，需要的技能也不一样。我没法替你决定你该装哪些，只能告诉你：如果你需要什么技能，去哪里找。

Find Skills 就是干这件事的，它可以帮你联网找你需要的技能。

它背后连着一个叫 skills.sh 的开放平台（Vercel 做的），各种人把自己写的 Skill 发布到上面。装上 Find Skills 之后，你直接跟 AI 说「我需要一个帮我写 PPT 的技能，帮我找一下」，它就会自动去平台搜索，找到合适的帮你一键安装。

那这个技能怎么装？

```
npx skills add https://github.com/vercel-labs/skills --skill find-skills
```

装好之后怎么用？

装完之后，直接跟 AI 说「我需要一个帮我做 XX 的技能，帮我找一下」就行。

第三个：Skill Creator——自己造技能包

前两个是用别人的经验。第三个是把你自己的经验也变成 Skill。

比如你写周报有自己的套路，或者做设计评审有一份检查清单，都可以用 Skill Creator 把它变成一个 Skill 文件。以后 AI 每次帮你做这件事都按你的方法来。还能分享给同事。

那这个技能怎么装？

```
npx skills add https://github.com/anthropics/skills --skill skill-creator
```

装好之后怎么用？

两种用法：

第一种：先让 AI 把活干了，干完之后跟它说「把刚才的做法总结成一个 Skill」，它会自动把这次对话里的流程、标准、偏好提炼成一个 Skill 文件，下次直接复用。

第二种：你脑子里有想法但不想自己动手整理，直接跟 AI 说「帮我写一个 XX Skill，用来处理 XX 问题」，它会帮你从零搭建。

个人推荐第一种。因为从实战中提炼出来的 Skill，比凭空想的靠谱得多。

最后，我推荐的三个 Skill 的逻辑是这样的：

先尝到 Skill 的味道 → 再去找更多 → 最后自己造

如果安装遇到问题？

如果你在执行命令时看到一堆报错，别慌。把报错信息直接复制粘贴发给 AI，它会告诉你怎么解决。大部分情况就是缺个环境或者网络问题，按 AI 给的提示操作就行。

AI 最擅长的事情之一就是看报错。这件事交给它就好。

写在后面

没有 Skill 的时候，你每次和 AI 对话都是从零开始。教过的东西下次还得教，踩过的坑下次还得踩。

有了 Skill，你是在积累的基础上继续。

我自己装了三十多个 Skill，写文章、做 PPT、管项目都有对应的。回不去了。
