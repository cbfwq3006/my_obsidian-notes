## 实际建议

|场景|推荐方法|
|---|---|
|快速摸底，5分钟出结果|相关性排序|
|日常分析，够用就行|随机森林|
|正式项目，需要深入解释|XGBoost + SHAP|

**我的建议是组合用**：先用相关性排序快速筛一遍 → 再用随机森林验证 → 两者结论一致的因素基本就是那20%的关键因。不一致的再用SHAP深挖。


XGBoost 告诉你：**"投诉次数"是流失的关键因素。**

但你怎么确认这不是数据偶然？这时需要统计检验：

python

复制

```python
from scipy import stats

# 卡方检验：分类变量 vs 流失（分类 vs 分类）
# 例如：会员等级 vs 是否流失
contingency = pd.crosstab(data['membership_level'], data['churn'])
chi2, p_value, dof, expected = stats.chi2_contingency(contingency)
print(f"卡方值={chi2:.2f}, p值={p_value:.4f}")
# p < 0.05 → 关系是真实的

# T检验：连续变量 vs 流失（连续 vs 分类）
# 例如：流失用户 vs 留存用户的平均登录天数有没有显著差异
churned = data[data['churn']==1]['login_days_30d']
retained = data[data['churn']==0]['login_days_30d']
t_stat, p_value = stats.ttest_ind(churned, retained)
print(f"T值={t_stat:.2f}, p值={p_value:.4f}")
```

### 完整流程应该是这样的

```undefined
第1步：相关性/XGBoost → 筛出疑似关键因素（海选）
第2步：统计检验 → 验证是否显著（确认不是巧合）  
第3步：SHAP → 解释具体怎么影响的（讲清楚机制）
```

**类比**：

- XGBoost = 侦探找到嫌疑人
- 统计检验 = 法庭上举证定罪（p < 0.05 = 排除合理怀疑）
- SHAP = 还原作案过程

少了统计检验这步，你的结论在业务汇报时容易被质疑："你怎么确定这不是偶然？"
