**接下来在 Claude Code 里，最稳妥的是用“分两步执行”**：先让它批量出单份报告，再让它汇总。

---

## 你现在就可以这样操作（推荐）

### 第1步：进入项目目录

在终端里先进入你的项目根目录：

bash

复制

```bash
cd party-review
```

---

### 第2步：在 Claude Code 里发这条指令（先做个人材料）

> 可直接复制下面整段：

text

复制

```text
请执行批量评分任务：
1) 读取 rubric/rubric-个人材料.md 作为唯一评分标准；
2) 遍历 materials/个人/ 下所有 .md 文件；
3) 对每份材料按Rubric逐维度评分（含子项），计算加权总分；
4) 检查硬伤项，并按规则得到最终得分；
5) 每份材料输出一个详细报告到 output/个人评分-原文件名.md；
6) 同时输出一个结构化JSON到 output/json/个人评分-原文件名.json；
7) 全部完成后生成汇总表 output/个人材料评分汇总.md，包含：
   - 文件名
   - 各维度分数
   - 加权总分
   - 硬伤项
   - 最终得分
   - 排名（按最终得分降序）
8) 若 output/json 目录不存在请自动创建；
9) 不要跳过任何文件，处理进度实时打印。
```

---

### 第3步：再发一条（班子材料）

text

复制

```text
请按同样流程处理班子材料：
- Rubric: rubric/rubric-班子材料.md
- 输入目录: materials/班子/
- 单份报告输出: output/班子评分-原文件名.md
- JSON输出: output/json/班子评分-原文件名.json
- 汇总输出: output/班子材料评分汇总.md
```

---

### 第4步：抽查结果

重点检查三类文件是否生成：

- `output/个人评分-xxx.md`（逐份详细报告）
- `output/json/个人评分-xxx.json`（机器可汇总）
- `output/个人材料评分汇总.md`（总排名）

---

## 一个实用建议（避免翻车）

你之前写的 `batch-review.sh` 用 `$(cat "$file")` 把全文塞进 prompt，材料很长时可能截断。  
在 Claude Code 里直接让它“读取文件路径并处理”更稳定（就是上面这种写法）。
