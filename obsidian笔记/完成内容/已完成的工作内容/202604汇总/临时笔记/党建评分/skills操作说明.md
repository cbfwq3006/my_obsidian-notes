### 1. 确定性

- 每次评分都按照相同的步骤执行

- 标准化的报告格式
- 统一的错误处理

### 3. 可维护性

- 评分标准变更只需修改 Skill 文件
- 版本控制更容易
- 团队协作更规范

### 4. 可追溯性

- 每份报告都包含评分标准版本号
- 评分时间戳
- 详细的评分依据

## 常见问题

**Q：Skill 文件和直接输入指令有什么区别？**

A：

- 直接指令：灵活但不稳定，每次可能执行方式略有不同
- Skill 文件：固定流程，确保每次评分完全一致

**Q：可以修改 Skill 文件吗？**

A：可以。如果你想调整评分流程、报告格式、错误处理方式，直接编辑 `党建材料评分.skill.yaml` 即可。

**Q：Skill 文件评分更慢吗？**

A：不会。Skill 文件只是规范了执行流程，实际评分速度相同。

**Q：如何查看 Skill 文件的执行日志？**

A：Claude Code 会在执行过程中显示每个步骤的进度。如果需要更详细的日志，可以在 Skill 文件中添加 logging 配置。
