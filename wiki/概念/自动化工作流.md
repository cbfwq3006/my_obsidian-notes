# 自动化工作流

## 概念定义

**自动化工作流**是指通过工具和脚本，将重复性的手动操作转变为自动执行的流程，减少人工干预，提高效率和一致性。

## 在知识管理中的应用

### 传统手动流程的痛点

1. **信息收集**：手动复制粘贴文章
2. **内容整理**：手动分类、打标签
3. **知识沉淀**：手动建立关联
4. **反馈查看**：需要主动打开查看

**问题**：
- 耗时耗力
- 容易遗漏
- 难以坚持
- 感觉像在给 AI 打工

### 自动化解决方案

**三步自动化流程**：
1. **自动抓取**：定时从互联网采集信息
2. **自动整理**：AI 编译成结构化知识
3. **自动汇报**：主动推送处理结果

## 完整自动化案例

### Hermes + AutoCLI + Obsidian 系统

#### 第一步：信息抓取自动化

**工具**：[[AutoCLI]] + [[Hermes]]

**流程**：
1. 安装 AutoCLI 的 skill
2. 配置抓取目标（如 X 上的点赞 top10）
3. 设置定时任务（如每天下午 5 点）
4. 自动创建多层级目录并归档

**价值**：解决"新信息怎么进来"

#### 第二步：内容整理自动化

**工具**：[[Hermes]] + LLM Wiki Skill

**流程**：
1. Hermes 读取抓取的内容
2. 调用 llm-wiki skill 转化为结构化知识
3. 自动分类、打标签、建立关联
4. 可设置定时自动执行

**价值**：让 Hermes 充当"知识编译器"

#### 第三步：反馈汇报自动化

**工具**：[[Hermes]] + 微信 Clawbot

**流程**：
1. 入库完成后自动生成日报
2. 包含：总览、同步结果、重点、摘要、位置
3. 自动发送到微信
4. 主动汇报督促使用

**价值**：建立反馈循环，让系统"活"起来

## 自动化的核心价值

### 1. 从"打工"到"被服务"

**之前**：
- 人工筛选文章
- 手动拷贝输入输出
- 充当 AI 的手脚
- 做枯燥的知识搬运

**之后**：
- 系统自动运转
- 自动接收、整理、汇报
- 人专注于思考和决策
- 知识库为我服务

### 2. 从"仓库"到"系统"

**被动仓库**：
- 我把内容往里堆
- 需要时手动搜索
- 时间长了就忘了

**主动系统**：
- 自动接新信息
- 自动归档整理
- 自动按主题沉淀
- 自动把结果汇报

### 3. 建立反馈循环

**关键洞察**：
- 很多知识库死在"没有反馈"
- 有沉淀但没有反馈
- 理论上在更新，但不会主动看
- 时间长了又变成被动仓库

**解决方案**：
- 每天自动发送日报
- 主动汇报处理结果
- 督促每天查看和使用
- 让"沉淀"变成"反馈"

## 其他自动化场景

### 自动阅读笔记

**配置**：
```bash
# 每周六上午10点阅读本周所有日记
hermes config set obsidian.read_schedule "0 10 * * 6"
hermes config set obsidian.read_scopes ["日记", "项目", "研究"]
```

**功能**：
- 定期扫描笔记
- 提取关键词和标签
- 生成知识摘要
- 提供关联建议

### 主动推送功能

**配置**：
```bash
hermes config set obsidian.proactive.enabled true
hermes config set obsidian.proactive.context_window 7
```

**功能**：
- 研究某话题时自动推送相关笔记
- 基于最近 7 天的上下文
- 避免重复思考

### 知识图谱自动更新

**功能**：
1. 自动扫描笔记中的双向链接
2. 提取关键词和标签
3. 生成知识摘要和关联建议
4. 写入图谱索引笔记

## 实现自动化的关键要素

### 1. 工具组合

- **抓取工具**：AutoCLI、Web Clipper
- **编译工具**：Hermes、LLM Wiki
- **存储工具**：[[Obsidian]]
- **反馈工具**：微信、邮件

### 2. 定时任务

- 固定时间执行
- 无需人工触发
- 持续运行

### 3. 反馈机制

- 主动汇报结果
- 可视化进度
- 督促使用

### 4. 可配置性

- 可调整抓取源
- 可定制整理规则
- 可修改汇报格式

## 系统感的价值

**表面上**：
- AutoCLI 负责抓
- Hermes 负责编
- Obsidian 负责看
- 微信负责报

**真正价值**：
- 四步被打通
- 形成完整闭环
- 知识库开始自己运转
- 从"堆积"到"经营"

## 最佳实践

### 1. 从小处开始

- 先自动化一个环节
- 验证效果后再扩展
- 逐步建立完整流程

### 2. 保留人工审核

- 自动化不等于完全无人
- 关键决策保留人工判断
- AI 辅助而非替代

### 3. 定期优化

- 观察自动化效果
- 调整参数和规则
- 持续改进流程

### 4. 建立反馈

- 自动化必须有反馈
- 否则变成"黑箱"
- 主动汇报保持可见性
