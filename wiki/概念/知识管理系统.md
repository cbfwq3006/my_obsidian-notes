# 知识管理系统

## 概念定义

知识管理系统是用于**收集、组织、存储和检索**知识的工具和方法论体系，目标是将碎片化信息转化为结构化知识，提升个人或团队的生产力。

## 核心问题

两篇文章都指出了知识管理的核心痛点：

### 人类记忆的局限
- 读过的文章很快忘记
- 研究过的问题需要重新查找
- 笔记写了很多但找不到
- 碎片化信息无法有效提取

**原因**：人类大脑擅长"模糊匹配"但不擅长"精确提取"

## 解决方案

### 方案 1：本地知识库（Obsidian + Hermes）

来自：[[wiki/来源/2026-04-12 用Hermes+Obsidian搭建个人知识库]]

**架构**：
```
Obsidian（存储）+ Hermes（检索）+ 人脑（思考）
```

**分工**：
- **Obsidian**：结构化、持久化、可视化存储
- **Hermes**：理解、关联、主动推送
- **人脑**：创造、决策、洞察

**特点**：
- 本地优先（数据在本地）
- 双向链接（形成知识网络）
- 图谱视图（可视化知识结构）
- 隐私安全

**文件夹结构**（PARA 方法）：
```
0_Inbox/        # 收集箱
1_Projects/     # 项目笔记
2_Areas/        # 领域笔记
3_Resources/    # 资源笔记
4_People/       # 人物笔记
5_Templates/    # 模板库
6_Meta/         # 元笔记
z_Archive/      # 归档
```

### 方案 2：云端知识库（飞书 + Codex）

来自：[[wiki/来源/2026-04-15 飞书CLI + Codex = 生产力]]

**架构**：
```
飞书知识库（存储）+ Codex（组织）+ 飞书CLI（连接）
```

**特点**：
- 云端协作（团队共享）
- 树状结构（层级组织）
- 超链接导航
- AI 辅助整理

**实践案例**：
- GDPR 中英文对照知识库
- 99 条法条的结构化组织
- 逐条验证确保准确性

## 两种方案对比

| 维度 | Obsidian + Hermes | 飞书 + Codex |
|------|------------------|--------------|
| **存储位置** | 本地 | 云端 |
| **适用场景** | 个人知识沉淀 | 团队协作 |
| **数据格式** | Markdown | 飞书文档 |
| **链接方式** | 双向链接 | 超链接 |
| **可视化** | 图谱视图 | 树状结构 |
| **隐私性** | 高（本地存储） | 中（云端存储） |
| **协作性** | 低 | 高 |
| **AI 集成** | Hermes | Codex |

## 混合方案

两篇文章暗示了一个**混合方案**的可能性：

```
飞书（团队协作）
    ↓
飞书 CLI（抓取）
    ↓
Obsidian（个人整理）
    ↓
Hermes（智能检索）
    ↓
飞书（分享回团队）
```

**优势**：
- 团队协作 + 个人沉淀
- 云端 + 本地双备份
- 结构化 + 网络化双组织

## 核心原则

### 1. 分工明确
- 工具各司其职，不追求"全能工具"
- 存储、检索、思考分离

### 2. 自下而上（Karpathy 原则）
- 概念从多篇来源中自然浮现
- 不对单篇文章过度抽象
- ≥2 篇来源才创建独立概念页

### 3. 质量优先
- 宁可多花时间，也要确保准确性
- 逐条验证 > 批量处理
- 人工监督 + AI 辅助

### 4. 渐进式积累
- 每天固定时间整理
- 及时摄取新内容
- 定期健康检查（lint）

## 工作流对比

### 日常研究流

**Obsidian 方案**：
1. 阅读文章 → Hermes 总结
2. 关键点 → 自动存入 Obsidian
3. 关联思考 → 指出与现有笔记的联系
4. 定期回顾 → 复习重要笔记

**飞书方案**：
1. PDF 文档 → markitdown 转换
2. Markdown → Codex 组织
3. 结构化内容 → 飞书 CLI 写入
4. 验证确认 → 逐条核对

### 项目管理流

**Obsidian 方案**：
1. 创建项目 → Hermes 生成模板
2. 日常更新 → 语音输入转笔记
3. 周报生成 → 汇总项目进度
4. 知识沉淀 → 自动归档

**飞书方案**：
1. 创建知识库 → 树状结构
2. 团队协作 → 多人编辑
3. 进度跟踪 → 飞书日历
4. 成果分享 → 飞书文档
