# Hermes+AutoCLI+Obsidian：自动化知识系统

**原始来源**: [[raw/Hermes+AutoCLI+Obsidian： 打造自动入库、自动整理、自动微信汇报的知识系统.md]]

## 核心观点

通过 Hermes Agent + AutoCLI + Obsidian 的组合，构建一个**自动抓取、自动整理、自动汇报**的知识系统，让知识库从"被动仓库"变成"主动系统"。

## 痛点分析

### 手动维护的问题

- 每天要手动筛选文章
- 手动拷贝输入输出
- 离不开电脑
- 感觉像在给 AI 打工，充当 AI 的手脚

### 解决方案

升级为全自动工作流：
1. 定时完成信息抓取
2. 信息编译进 LLM Wiki
3. 微信汇报入库结果

## 三步流程详解

### 第一步：Hermes + AutoCLI - 信息抓取自动化

**AutoCLI 的优势**：
- 无需 API KEY
- 复用 Chrome 浏览器已有登录态
- 支持 90+ 站点信息抓取

**操作流程**：
1. 安装 AutoCLI 的 skill
2. 在 Obsidian 建立 AI NEWS HUB 文件夹
3. 让 Hermes 执行内容抓取（如 X 上点赞 top10）
4. Hermes 自动创建多层级目录，按日期归档
5. 创建定时任务，每天下午 5 点自动执行

**核心价值**：解决"新信息怎么进来"的问题。

### 第二步：Hermes + LLM Wiki Skill - 定时梳理入库

**核心问题**：进来的信息怎么变成可复用的知识，而不是堆成素材仓库？

**操作流程**：
1. 让 Hermes 把抓取的文章转化为 LLM Wiki
2. Hermes 自动调用内置的 llm-wiki skill
3. Wiki 结构可二次调整
4. 可设置定时自动执行

**核心价值**：让 Hermes 充当**知识编译器**，把信息流压成认知地图。

### 第三步：Hermes + 微信 Clawbot - 微信日报汇报

**核心问题**：知识库在后台更新，但没有反馈机制，时间长了又变成被动仓库。

**解决方案**：每天同步完成后，自动发送微信日报。

**日报内容**：
- 今日总览一句话
- 同步结果
- 今日重点
- 100 字摘要
- 报告位置

**操作流程**：
1. 让 Hermes 接入个人微信渠道
2. 扫描二维码连接微信
3. 串联前两步任务
4. 入库完成后自动生成日报并发送

**核心价值**：主动汇报督促每天查看和使用知识库，让"沉淀"变成"反馈"。

## 系统感的价值

### 表面上的分工

- AutoCLI 负责抓
- Hermes 负责编
- Obsidian 负责看
- 微信负责报

### 真正的价值

四步被打通，知识库开始自己运转：
- 自动接新信息
- 自动归档整理
- 自动按主题沉淀
- 自动把结果汇报

**从"我给知识库打工"变成"知识库为我服务"**。

## 个人思考

这篇文章展示了如何通过工具组合实现知识管理的全流程自动化。关键不在于单个工具的强大，而在于打通整个链路：抓取 → 整理 → 沉淀 → 反馈。最后一步"微信日报"看似最轻，实际最关键，因为它建立了反馈循环，让系统真正"活"起来。
