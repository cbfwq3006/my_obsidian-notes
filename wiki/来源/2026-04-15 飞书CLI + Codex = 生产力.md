# 飞书CLI + Codex = 生产力

> 原文：[[raw/飞书CLI + Codex = 生产力]]

## 核心观点

本文介绍了如何使用飞书 CLI + Codex 构建 GDPR 中英文对照知识库的实践案例，展示了 AI Agent 与飞书集成的生产力提升效果。

**核心流程**：
1. 安装飞书 CLI 和 skills
2. 使用 markitdown 将 PDF 转换为 Markdown
3. 用 Codex 在飞书知识库中创建树状结构
4. 逐条插入中英文对照内容

## 关键要点

### 1. 飞书 CLI 安装与配置

**安装命令**：
```bash
npm install -g @larksuite/cli
npx skills add https://github.com/larksuite/cli -y -g
```

**效果**：
- 安装后会在 Trae、Cursor、Codex、Claude Code 等应用的 skills 文件夹中新增飞书相关 skill
- AI Agent 可以通过 lark-cli 访问飞书资源

**初始化**：
```bash
lark-cli config init --new
```
- 执行后会生成链接
- 完成初始化后，飞书开发者平台会创建新应用

**授权**：
```bash
lark-cli auth login
```
- 进入交互式界面
- 可选择授权的业务域（文档、知识库、云空间等）

**精确授权**：
```bash
lark-cli auth login --domain docs
```

### 2. 实践案例：GDPR 知识库

**背景**：
- GDPR 正文 56 页，11 章节，99 条法条，全英文
- 中英文在两个 PDF 中，阅读不便
- 目标：创建中英文逐条对照的知识库

**工具链**：
1. **markitdown**：PDF 转 Markdown（1 分钟完成）
2. **Codex**：在飞书知识库中组织内容
3. **飞书 CLI**：连接 Codex 和飞书

**实施步骤**：

**步骤 1**：让 Codex 创建树状结构
- 将法条逐一放进 GDPR 知识库
- 建立层级关系

**步骤 2**：添加超链接
- 为每个文档增加导航链接
- 方便跳转查阅

**步骤 3**：插入中文翻译（最关键）
- **不是一次性导入**，而是逐条插入
- 每插入一条，回读核对
- Codex 会主动停下来确认
- 用了 4 个大对话才完成
- **非常费 token**

### 3. 关键经验

**谨慎策略**：
- 对于需要高准确率的任务，采用逐条处理而非批量处理
- 每步都要验证，确保正确性
- AI 会主动寻求确认，这是好事

**工具选择**：
- markitdown：PDF 转换神器
- Codex：适合结构化内容组织
- 飞书知识库：适合存储和展示

**成本考虑**：
- 高质量输出需要大量 token
- 4 个大对话完成 99 条法条的处理

### 4. 成果

- 完成了 GDPR 中英文对照知识库
- 阅读和理解更加方便
- 树状结构 + 超链接 + 双语对照
