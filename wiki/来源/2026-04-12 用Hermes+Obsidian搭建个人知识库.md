# 用Hermes+Obsidian搭建个人知识库

**原始来源**: [[raw/用Hermes+Obsidian搭建个人知识库：做你的"第二大脑".md]]

## 核心观点

文章提出用 Obsidian + Hermes 构建"第二大脑"的解决方案，解决人类记忆的局限性问题。核心思想是：**Obsidian 负责存储，Hermes 负责检索和整理，人脑负责思考**。

## 关键要点

### 为什么需要第二大脑

- 人类大脑擅长"模糊匹配"但不擅长"精确提取"
- 读过的内容容易遗忘，研究过的问题需要重新查找
- 笔记写了很多但要用时找不到

### Obsidian 的五大优势

1. **本地优先**：数据存在本地 Markdown 文件，永不丢失
2. **双向链接**：笔记互相链接，形成知识网络
3. **图谱视图**：可视化知识结构
4. **完全可定制**：CSS、插件、模板可自定义
5. **隐私安全**：数据在本地，不在云端

### Hermes 的四大优势

1. **主动记忆**：自动把重要信息写入 Obsidian
2. **智能检索**：用自然语言查询知识库
3. **自动整理**：帮助分类、标签、建立关联
4. **持续学习**：每次对话强化知识网络

### 推荐的文件夹结构

```
~/obsidian-vault/
├── 0_Inbox/        # 收集箱
├── 1_Projects/     # 项目笔记
├── 2_Areas/        # 领域笔记
├── 3_Resources/    # 资源笔记
├── 4_People/       # 人物笔记
├── 5_Templates/    # 模板库
├── 6_Meta/         # 元笔记
└── z_Archive/      # 归档
```

### Hermes + Obsidian 联动配置

1. 安装 **Local REST API** 插件
2. 配置 Hermes 连接 Obsidian
3. 创建 Obsidian 操作 Skill
4. 实现自动保存、自动阅读、主动推送功能

## 实战工作流

1. **日常研究流**：阅读 → Hermes 总结 → 自动存入 Obsidian → 关联思考 → 定期回顾
2. **项目管理流**：创建项目 → 日常更新 → 周报生成 → 知识沉淀
3. **读书笔记流**：读书 → 提取内容 → 生成书摘 → 输出实践清单

## 效果对比

| 指标 | 使用前 | 使用后 |
|------|--------|--------|
