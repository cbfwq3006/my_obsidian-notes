# 用Hermes+Obsidian搭建个人知识库

> 原文：[[raw/用Hermes+Obsidian搭建个人知识库：做你的"第二大脑"]]

## 核心观点

本文介绍了如何用 Obsidian + Hermes 搭建个人知识管理系统，解决"读过就忘"、"笔记找不到"等问题。

**核心理念**：
- Obsidian 负责"存储"（结构化、持久化、可视化）
- Hermes 负责"检索"（理解、关联、主动推送）
- 人脑负责"思考"（创造、决策、洞察）

## 关键要点

### 1. 为什么需要第二大脑

**问题**：
- 读过的文章很快忘记
- 研究过的问题需要重新查找
- 笔记写了很多但找不到
- 碎片化信息无法有效提取

**原因**：人类大脑擅长"模糊匹配"但不擅长"精确提取"

### 2. Obsidian 的优势

1. **本地优先**：所有笔记存在本地 Markdown 文件
2. **双向链接**：笔记之间互相链接，形成知识网络
3. **图谱视图**：可视化知识结构
4. **完全可定制**：CSS、插件、模板可自定义
5. **隐私安全**：数据在本地硬盘

### 3. Hermes 的优势

1. **主动记忆**：自动把重要信息写入 Obsidian
2. **智能检索**：用自然语言查询知识库
3. **自动整理**：帮助分类、标签、建立关联
4. **持续学习**：每次对话强化知识网络

### 4. 推荐插件

| 插件 | 作用 | 重要性 |
|------|------|--------|
| Templater | 模板系统，自动化创建笔记 | ⭐⭐⭐⭐⭐ |
| Dataview | 查询笔记，像数据库一样 | ⭐⭐⭐⭐⭐ |

| Metaedit | 可视化编辑 YAML | ⭐⭐⭐⭐ |

### 5. 文件夹结构设计

```
~/obsidian-vault/
├── 0_Inbox/        # 收集箱
├── 1_Projects/     # 项目笔记
├── 2_Areas/        # 领域笔记
├── 3_Resources/    # 资源笔记
├── 4_People/       # 人物笔记
├── 5_Templates/    # 模板库
├── 6_Meta/         # 元笔记（索引/地图）
└── z_Archive/      # 归档
```

### 6. Hermes + Obsidian 联动配置

**步骤**：
1. 在 Obsidian 中安装 **Local REST API** 插件
2. 开启"允许来自 LAN 的请求"
3. 配置 Hermes：
   - 设置 vault 路径
   - 启用 API
   - 设置端口（27123）

### 7. 高级功能

**自动阅读笔记**：
- 配置 Hermes 定期阅读笔记
- 设置阅读范围（日记、项目、研究）

**知识图谱自动更新**：
- 自动扫描双向链接
- 提取关键词和标签
- 生成知识摘要和关联建议

**主动推送**：
- 研究某话题时，Hermes 主动推送相关笔记
- 设置上下文窗口（如 7 天内的相关笔记）

### 8. 实战工作流

**日常研究流**：
1. 阅读文章 → Hermes 总结
2. 关键点 → 自动存入 Obsidian
3. 关联思考 → 指出与现有笔记的联系
4. 定期回顾 → 复习重要笔记

**项目管理流**：
1. 创建项目 → 生成项目笔记模板
2. 日常更新 → 语音输入转笔记
3. 周报生成 → 汇总项目进度
4. 知识沉淀 → 项目结束后自动归档

**读书笔记流**：
1. 读书 → 重要段落拍照/复制
2. 上传 → 提取关键内容
3. 整理 → 自动生成书摘笔记
4. 输出 → 写读后感/实践清单

## 个人思考

这篇文章提出的方法论非常实用，核心是"分工明确"：
- Obsidian 做持久化存储
- Hermes 做智能检索和整理
- 人脑专注于创造性思考

这种架构避免了"全能工具"的陷阱，让每个工具发挥自己的优势。
