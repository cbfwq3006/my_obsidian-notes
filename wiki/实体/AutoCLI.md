# AutoCLI

## 基本信息

**类型**：浏览器信息抓取工具
**全称**：OpenCLI / AutoCLI
**特点**：开源项目

## 核心优势

### 1. 无需 API KEY
- 直接复用 Chrome 浏览器已有的登录态
- 不需要额外配置 API
- 降低使用门槛

### 2. 广泛的站点支持
- 支持 90+ 站点的信息抓取
- 包括社交媒体、新闻网站等
- 持续扩展中

### 3. 与 Hermes 的集成
- 可作为 Hermes 的 skill 安装
- 让 Hermes 接管内容抓取
- 实现自动化信息采集

## 使用场景

### 信息抓取自动化
- 抓取 X (Twitter) 上的点赞 top10 推文
- 抓取特定主题的文章
- 按日期自动归档

### 与 Obsidian 的配合
1. 在 [[Obsidian]] 中建立目标文件夹（如 AI NEWS HUB）
2. 让 [[Hermes]] 在该路径下执行内容抓取
3. AutoCLI 自动创建多层级目录
4. 按日期归档抓取内容

### 定时任务
- 设置每天固定时间执行抓取
- 如每天下午 5 点抓取当日热门内容
- 完全自动化，无需人工干预

## 在自动化工作流中的角色

**信息入口层**：
- 解决"新信息怎么进来"的问题
- 自动从互联网采集信息
- 为后续整理提供原始素材

## 与其他工具的协作

### AutoCLI + Hermes + Obsidian
1. **AutoCLI**：负责抓取信息
2. **Hermes**：负责编译整理
3. **Obsidian**：负责存储展示

形成完整的自动化知识入库流程。

## 核心价值

- 把人从重复的信息搬运工作中解放出来
- 实现信息采集的自动化
- 为知识管理提供持续的信息源
