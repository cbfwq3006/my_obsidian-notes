# 飞书

## 基本信息

**类型**：企业协作平台
**开发商**：字节跳动
**核心功能**：即时通讯、文档协作、知识库、日历、视频会议

## 在知识库中的角色

飞书在本知识库的两篇文章中都作为**知识存储和协作平台**出现。

### 第一篇文章中的角色

[[wiki/来源/2026-04-12 用Hermes+Obsidian搭建个人知识库]]

- 作为**团队协作平台**
- 与 Obsidian 形成互补：
  - Obsidian：个人知识沉淀
  - 飞书：团队协作
- 通过 [[wiki/概念/飞书CLI]] 与 Hermes 集成

**集成方案**：
- 飞书文档导出到 Obsidian 做深度整理
- Obsidian 精华内容分享回飞书

### 第二篇文章中的角色

[[wiki/来源/2026-04-15 飞书CLI + Codex = 生产力]]

- 作为**知识库平台**
- 存储 GDPR 中英文对照内容
- 通过 [[wiki/概念/飞书CLI]] 与 [[wiki/实体/Codex]] 集成

**使用场景**：
- 创建树状知识库结构
- 存储 99 条法条的双语对照
- 提供超链接导航

## 核心产品

### 1. 飞书文档
- 在线协作文档
- 支持 Markdown
- 可导出多种格式

### 2. 飞书知识库
- 树状结构组织
- 支持超链接
- 适合构建知识体系

### 3. 飞书云空间
- 文件存储
- 版本管理
- 权限控制

### 4. 飞书 CLI
详见：[[wiki/概念/飞书CLI]]

## 与其他工具的集成

### 与 Obsidian 集成
- **方向 1**：飞书 → Obsidian（深度整理）
- **方向 2**：Obsidian → 飞书（团队分享）
- **工具**：飞书 CLI、Hermes

### 与 AI Agent 集成
- **支持的 AI 工具**：Trae、Cursor、Codex、Claude Code
- **集成方式**：通过 skills 调用 lark-cli
- **能力**：读写文档、管理知识库、发送消息

## 优势

1. **一体化平台**：文档、通讯、日历集成
2. **开发者友好**：提供 CLI 和 API
3. **AI 原生支持**：官方支持 AI Agent 集成
4. **权限管理**：精细的权限控制

## 使用场景总结

| 场景 | 工具组合 | 用途 |
|------|---------|------|
| 个人知识管理 | 飞书 + Obsidian + Hermes | 团队协作 + 个人沉淀 |
| 知识库构建 | 飞书 + Codex + 飞书 CLI | 结构化内容组织 |
| 文档处理 | 飞书 + markitdown | PDF 转换和存储 |
