# Codex

## 基本信息

**类型**：AI 编程助手
**核心能力**：代码生成、文档处理、结构化内容组织

## 在知识库中的角色

Codex 在两篇文章中都作为 **AI Agent** 出现，用于自动化知识管理任务。

### 第一篇文章中的提及

[[wiki/来源/2026-04-12 用Hermes+Obsidian搭建个人知识库]]

- 在推荐插件列表中被提及
- 与 [[wiki/概念/飞书CLI]] 集成
- 安装飞书 CLI 后，Codex 的 skills 文件夹会新增飞书相关 skill

### 第二篇文章中的核心角色

[[wiki/来源/2026-04-15 飞书CLI + Codex = 生产力]]

Codex 是本文的**主角**，展示了其在知识库构建中的强大能力。

**具体任务**：
1. **创建树状结构**：将 99 条 GDPR 法条组织成知识库
2. **添加超链接**：为每个文档增加导航链接
3. **插入翻译**：将中文翻译精准插入到英文法条下方

**工作特点**：
- **谨慎性**：时不时停下来寻求用户确认
- **渐进式**：逐条处理而非批量处理
- **高准确率**：通过回读核对确保正确性

**成本**：
- 处理 99 条法条用了 **4 个大对话**
- **非常费 token**

## 核心能力

### 1. 结构化内容组织
- 创建层级结构
- 建立文档关联
- 添加导航链接

### 2. 精确内容处理
- 逐条插入内容
- 回读验证
- 主动寻求确认

### 3. 与飞书 CLI 集成
- 通过 skills 调用 lark-cli
- 直接操作飞书知识库
- 无缝读写文档

## 使用场景

### 场景 1：GDPR 知识库构建

**背景**：
- 56 页英文法规文档
- 需要中英文对照
- 要求高准确率

**工具链**：
markitdown（PDF 转换）→ Codex（内容组织）→ 飞书 CLI（写入知识库）

**策略**：
- 不一次性导入，而是逐条处理
- 每插入一条，回读核对
- Codex 主动停下来确认
- 确保准确性优先于速度

### 场景 2：知识管理自动化

与其他工具配合：
- **Codex + 飞书**：云端知识库
- **Hermes + Obsidian**：本地知识库
- **Codex + 飞书 CLI**：自动化处理

## 优势

1. **谨慎可靠**：会主动寻求确认
2. **结构化能力强**：擅长组织复杂内容
3. **集成友好**：支持飞书 CLI skills
4. **精确处理**：适合需要高准确率的任务

## 局限性

1. **Token 消耗大**：复杂任务费 token
2. **速度较慢**：谨慎策略导致处理时间长
3. **需要人工监督**：不能完全自动化

## 与其他 AI 工具对比

| 工具 | 特点 | 适用场景 |
|------|------|---------|
| Codex | 谨慎、结构化 | 高准确率任务、知识库构建 |
| Hermes | 主动记忆、智能检索 | 个人知识管理、Obsidian 集成 |
| Claude | 对话理解强 | 复杂推理、内容创作 |

## 实践经验

### 来自第二篇文章的经验

**质量优先策略**：
- 宁可多花 token，也要确保准确性
- 逐条处理 > 批量处理
- 每步验证 > 一次完成

**适用任务**：
- 法律文档处理
- 技术文档翻译
- 知识库构建
- 需要高准确率的内容组织

**不适用任务**：
- 简单重复性工作（太费 token）
- 不需要验证的批量处理
