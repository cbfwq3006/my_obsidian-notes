辉哥奇谭 *2026年4月19日 07:03*

两周前我还觉得 Hermes 不过如此。一个 OpenClaw 的模仿者，做得再像，生态也追不上。OpenClaw 有那么多开发者，有群众基础，有厂商支持，遥遥领先。

两周后我被打脸了。

先说个冷知识。Hermes 这个词，h 不发音，末尾的 s 要发音，念起来像「air 密斯」——对，就是爱马仕的法语发音。这个名字火起来不是因为奢侈品，而是因为它是一个 Agent 平台，很多功能跟 OpenClaw 非常像，但有一些地方，让我从「不以为然」变成了「真香」。

转机发生在上周末。

我花了两天时间，按照 Karpathy 大神的方法，用大语言模型把自己过去所有的文档、笔记、一本书的电子版，全部变成了一个 wiki。这个过程烧了 15 亿 Token，处理了 3000 多篇文章和 1 万多条笔记。处理完之后，生成了几百个新的 MD 文件，打开 Obsidian 在这些 wiki 之间来回跳转，感觉就像在翻自己写的「原则」——瑞·达利欧那种。

但光有知识库是死的。

我想把它变活。于是我用飞书连上 Hermes Agent，跟它说：我有一个 wiki，在某个目录下，你去看看。它看完之后说：我知道你的 wiki 在这里。然后我就基于这个 wiki 开始对话。

对话一段时间后我说：能不能直接生成一个 Skill，专门用来跟这个知识库对话？它就生成了。

然后我又说：能不能把我们对话中产生的结论，存回 wiki？每次对话都在增强这个 wiki？

它做到了。

其实到这一步，大家可能觉得也没什么神奇。OpenClaw 理论上都能做。但问题在于：Hermes 做这些事情的过程，非常顺畅、非常丝滑。

这话是对着 OpenClaw 说的。

OpenClaw 经常涌现出一些智能，让你觉得它挺懂我的，挺聪明的。但过一会儿它又不聪明了——刚教的东西又忘了，刚说好存在这里又跑去那里存了。Hermes 这种情况出现的概率，明显要低。

我总结了 Hermes 的三个突出优点。

第一，开箱即用。装好之后配好大模型就能用。不像 OpenClaw，还要设 user、设 soul、设各种东西。那些配置是 OpenClaw 的优势，但同时也是 OpenClaw 的门槛。对于想在严肃生产力环境下用 Agent 的人来说，简单、稳定，比什么都重要。

第二，自我迭代升级。这是我特别想强调的。在用 Hermes 的过程中，它经常主动提示：这件事我可以把它变成 Skill，你以后可以复用。越用它越聪明，这不是客气话。

我在这里要特别说一件事：2026 年有两个关键词，一个是 Agent，另一个是自我迭代更新。这两件事加在一起，会变成 2026 年的主旋律。Hermes 在自我迭代这件事上，让我体验到了「越用越爽」是什么感觉。

第三，能干活，有生产力。用 OpenClaw 想用出生产力，其实挺难的。三天两头出问题，记忆力也不行，做好的 Skill 有时候也不被唤起。我最后用 OpenClaw 用得最多的事情，就是让它帮我记笔记、查笔记，光这件事就花了很长时间。

一个以前在百度、现在在字节的老朋友跟我说了句话，我觉得很到位：OpenClaw 到最后更多的是情感化的——用户跟那个大龙虾之间是一种养成系的交流。真正把 OpenClaw 用于生产力的，其实很少。

Hermes 不一样。它带着一种可以变成生产力的潜力。

怎么装呢？网上搜到官网就行了。更简单的方式：让你的 Coding Agent 去装。无论是 Claude Code、Codex、千问 Coder 还是 OpenCode 还是 Codex，你只需要准备一个大模型的 Key。

最后说回「知识库变活」这件事。

Karpathy 强调的 LLM wiki 不是死的——你可以不断跟它对话，不断提问。但如果只是在电脑上跟构建知识库的那个 Coding Agent 提问，未免太死板。我把 Hermes Agent 接上飞书之后，随时随地都能跟自己的知识库对话。而且每次对话都在让知识库变得更好。

这件事的意义是什么？知识不再是存档，而是活的系统。每一次提问都是一次投资，每一次回答都让资产增厚一层。

如果你还没试过用 Agent 构建自己的知识库，现在是开始的时候。如果你试过了但觉得不够好用，试试 Hermes。

我说这话的底气是——两周前我还看不上它，但现在的我，已经爱上了它。
