## 🚀 快速开始

### 1. 初始化飞书通知

```bash
./scripts/feishu-notify.sh init
```

这会：
- 获取你的飞书用户信息
- 保存配置到 `~/.obsidian-feishu-config`
- 测试通知功能

### 2. 测试飞书通知

```bash
./scripts/feishu-notify.sh test
```

你应该会收到一条测试消息。

### 3. 从飞书抓取内容

```bash
./scripts/auto-workflow.sh fetch https://your-domain.feishu.cn/docx/xxxxx
```

内容会保存到 `raw/` 目录。

### 4. 整理到知识库

告诉 Claude：
```
摄取 raw/2026-05-04_文档标题.md
```

Claude 会：
- 分析文档内容
- 创建 wiki 来源摘要页
- 提取实体和概念（如果在多篇文章中出现）
- 更新 `wiki/index.md`
- 记录到 `wiki/log.md`

## 📖 使用场景

### 场景 1：从飞书文档学习

**步骤**：
1. 在飞书中找到想学习的文档
2. 复制文档链接
3. 运行：`./scripts/auto-workflow.sh auto <链接>`
4. 等待 Claude 整理完成
5. 查看 `wiki/index.md` 了解新增内容

### 场景 2：日常阅读积累

**步骤**：
1. 使用 Obsidian Web Clipper 保存网页到 `raw/`
2. 告诉 Claude："摄取 raw/最新文章.md"
3. Claude 自动整理到 wiki
4. 收到飞书通知

### 场景 3：知识查询

**步骤**：
1. 告诉 Claude："查询关于 XXX 的内容"
2. Claude 会搜索 `wiki/` 目录
3. 综合多个来源给出答案
4. 标注来源页面

### 场景 4：定期回顾

**步骤**：
1. 告诉 Claude："检查 wiki" 或 "lint"
2. Claude 会检查：
   - 矛盾的说法
   - 孤儿页面
   - 缺失的链接
   - 过时信息
3. 自动修复问题

## 🛠️ 脚本说明

### auto-workflow.sh

主自动化脚本，支持以下命令：

```bash
# 从飞书抓取
./scripts/auto-workflow.sh fetch <飞书URL>

# 整理到 wiki（会提示你告诉 Claude）
./scripts/auto-workflow.sh ingest <文件路径>

# 完整自动化流程
./scripts/auto-workflow.sh auto <飞书URL>

# 发送通知
./scripts/auto-workflow.sh notify "消息内容"
```

### feishu-notify.sh

飞书通知脚本：

```bash
# 初始化配置
./scripts/feishu-notify.sh init

# 发送消息
./scripts/feishu-notify.sh send "你好"

# 发送知识库更新通知
./scripts/feishu-notify.sh wiki-update "raw/文件.md" "摘要内容"

# 测试
./scripts/feishu-notify.sh test
```

## 🔧 高级配置

### 配置文件位置

- **飞书 CLI 配置**：`~/.lark-cli/config.json`
- **通知配置**：`~/.obsidian-feishu-config`
- **Claude 配置**：`CLAUDE.md`

### 自定义通知内容

编辑 `scripts/feishu-notify.sh` 中的 `send_wiki_update` 函数。

### 添加定时任务

使用 cron 定时执行：

```bash
# 编辑 crontab
crontab -e

# 每天下午 5 点检查 wiki 健康度
0 17 * * * cd /path/to/vault && echo "检查 wiki" | your-claude-command
```

## ❓ 常见问题

### Q1: 飞书通知发送失败？

**A**: 检查权限配置：

```bash
# 重新登录授权
lark-cli auth login --scope "im:message:send_as_bot"

# 重新初始化
./scripts/feishu-notify.sh init
```

### Q2: Claude 找不到文件？

**A**: 确保使用相对路径：

```
✓ 正确：摄取 raw/文件.md
✗ 错误：摄取 /Users/abc/.../raw/文件.md
```

### Q3: 如何批量处理多个文档？

**A**: 使用循环：

```bash
for file in raw/*.md; do
    echo "摄取 $file" | your-claude-command
done
```

### Q4: 概念页什么时候创建？

**A**: 根据 Karpathy 原则，只有当同一概念在 **≥2 篇不同来源** 中出现时才创建独立概念页。
